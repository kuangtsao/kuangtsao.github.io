alphacamp simple twitter 協作心得

發表於 2022-06-03

前言

Alphacamp 的最後一個階段的多人協作專案，simple twitter，目的是藉由打造一個簡單的 twitter 網站，以達到多人協作的開發目的；在一開始找組別時，非常感謝劉明與 ruby 找我一組，沒想到最大的煩惱就在一開始解決了。

專案介紹

github repo

專案 demo

前端： css, js, bootstrap, handlebars

後端: nodejs 的 express 框架，依照 RESTful 的方式進行路由設計。

利用 Notion 的功能進行文件分類與進度控管
Discord 作為主要的交流方式

主要負責功能

  • 當大家的 git master
    • 引入 github flow
    • 帶組員操作 git 功能，協作上不煩惱
    • github 的使用體驗改善
    • 引導解 conflict，以及讓組員能夠自行 merge code 到 master 分支
  • 專案的基礎建設
  • 推文與回復功能
  • 編輯會員資料與編輯個人主頁功能
  • api 版編輯功能

心得

首先非常感謝 ruby 一路建設好文件的架構，並且清楚說我們的進度在哪裡，還有說出很重要的一件事 「吵架了也不要消失」，但在過程之中也沒吵過半次；另外劉明在我眼中也是就有種必殺仕事人的感覺，感覺功能都難不倒他。

由於我們很早開始進行，相對其他組，我們比較有充裕的時間進行各個 sprint，開發過程大部分來講也都是順暢的；困擾的前端畫面也在劉明的極大貢獻，至少還是有著一個型出來。

途中我也利用著前份工作經驗，帶領兩位比較沒有經驗的夥伴利用 git 一起進行協作，目前觀察下來我們組應該是協作起來蠻順暢的。兩位組員到了後期，面對 conflict 和自行 merge 到 master 沒有那麼大的心靈壓力。

在此我體驗到了與他人協作開發會是什麼樣的狀況，非常感謝兩位組員的協作與付出！
