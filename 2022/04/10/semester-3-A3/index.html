發表於 2022-04-10

為何會選擇這個專案？

這個專案是對之前學到的技術的總體檢，裡面包含了 bootstrap 的應用，資料庫與 api 設計，CRUD 實作以及使用者認證
我覺得將學到技術再組合成一個解決方案是很有挑戰性的，同時可以檢驗自己的是否學的扎實。

使用了什麼技術？

前端利用了 bootstrap 與 font-awesome 做成了基本畫面，並利用 express 中的 partial view 功能與 connect-flash 做結合，達到推送錯誤訊息給使用者的效果。

後端方面：利用 passport 的本地策略達成了使用者認證，並利用 bcrypt 對敏感資料進行雜湊；
資料取得方面，利用 moogoose 和 async/await 做搭配，以達到每個 database opreation 都能確定執行完的效果。

哪部分你相對能掌握？哪裡花了最多時間？

相對能掌握的部份應當是資料庫與 api 設計，以及實現後端功能部分；建立種子資料的部分花了比較的時間，考慮到先後順序以及原始資料的處理花了稍微長的時間。

過程中碰到什麼困難？又如何克服？（例如：查找網路文件）

該專案由於有時效性，所以放棄了一些規格沒有提到的功能，以 MVP 的方式進行開發。

過程中你有對哪個技術有特別深刻的學習？

async/await，在 moogoose 的官方網站上有提到搭配起來正確的用法，並將其應用在需要的場合。
